# 要件定義

## 目的

PRIVATE SAUNA KONOSU のアイドルタイム枠を、ブランド価値を下げずに招待制で販売する。提携先企業が持つ優良顧客データを利用し、VIP 条件を満たす顧客だけにシークレット予約枠を公開する。

## 利用者

| 利用者 | できること |
|---|---|
| 一般会員 | 会員証、お知らせ、マイページ確認 |
| VIP 会員 | シークレット枠の予約、予約確認、履歴確認 |
| 管理者 | VIP 抽出、予約枠、予約、お知らせ、紹介コードの管理 |
| 提携先 | 連携顧客、VIP 判定、予約成立、紹介料の確認 |

## 機能要件

- 提携先企業と紹介コードを管理できる
- 提携先顧客データを保存できる
- 顧客ごとに VIP ランクと招待状態を保存できる
- シークレット予約枠を作成、公開、非公開にできる
- VIP 会員の予約を DB に保存できる
- 予約登録時に決済状態と紹介料を保存できる
- お知らせを作成し、会員画面へ表示できる
- フロントエンドの初期表示データを API から取得できる

## 非機能要件

- SQLite でローカル起動できる
- Java 標準 HTTP サーバーで静的ファイルと API を配信する
- DB テーブルはデモに必要な 4 つに抑える
- 旧スキーマの DB が残っている場合は初回起動時に最小構成へ作り直す
- サンプルデータは繰り返し初期化しても重複しない
